# Link World 产品文档

## 1. 项目概述（Project Overview）

Link World 是一个基于区块链技术的全球去中心化通信服务平台，连接运营商与 Web3 用户，提供跨境流量与通话服务。

使命：让全球用户无论身处何地，都能以优惠、便捷的方式获得本地化流量和通话服务，彻底解决传统漫游费用高、套餐不灵活、短期跨境使用痛点。

核心定位：Web3 时代的普惠通信网络 —— 无需 KYC、无需传统银行账户，仅用钱包地址即可实现"先用后付"的全球无缝通信。

技术栈亮点：

- 0G Chain：部署核心智能合约（计费、保证金、结算）。
- 0G Storage：去中心化存储使用记录、账单和 AI 模型。
- 0G Compute：驱动 AI 流量预测、动态推荐和异常检测。
- 0G Data Availability：确保所有通信数据可验证。
- Chainlink Oracle：提供可靠的实时外部数据输入，包括汇率转换、运营商定价参考和跨链消息传递。
- Chainlink CCIP：支持未来多链扩展和运营商系统跨链交互。
- Chainlink Data Streams：低延迟数据馈送，用于动态计费和实时汇率更新（与 0G 原生集成完美匹配）。

## 2. 问题与痛点（Problem Statement）

- 国际漫游费用高昂且不透明。
- 短期旅行/工作用户难以快速切换本地网络。
- 传统后付费需要 KYC 和银行绑定，门槛高。
- 跨币种结算时汇率波动大，缺乏可靠的链上定价机制。
- 运营商数据与区块链系统之间存在信任和数据同步难题。

## 3. 解决方案（Solution）

Link World 构建去中心化、AI + Oracle 增强的通信服务层，用户通过钱包无缝接入全球运营商网络。

核心功能模块：

### 3.1 先用后付（Post-Paid with Deposit）

- 用户授权钱包 + 邮箱，预存基础保证金即可使用。
- 智能合约实时监测流量与通话使用（由 0G Compute + 0G Storage 支持）。
- 月底（UTC 12:00）自动生成账单并通过邮箱发送。
- 用户成功存入押金后，其账户状态变更为"活跃"，方可申请服务。
- 押金可接受主流加密货币（USDT 或等值 ETH/BTC），优先使用稳定币。
- 支持随时手动结算，可统一月底结算；逾期 2 周将扣除保证金。

用户需满足以下条件方可提取押金：

1. 已结清所有账单。
2. 在最近一次结清账单后，未再次产生任何服务使用费用。
3. 用户主动发起押金提取请求，合约在验证条件满足后，自动解锁并退还押金至用户钱包。

### 3.2 不同地区便捷转网（Seamless Regional Switching）

- 选择目标地区 → 提交保证金 → 获得虚拟号码与接入凭证（支持 eSIM、虚拟号码、设备直连）。
- 国家选择：用户可在界面选择目标国家/地区，查看该地区合作运营商提供的服务。
- 虚拟号码申请：用户选择国家与运营商后，平台为其分配一个虚拟号码。用户获得虚拟号码及相应的接入密码或配置信息（例如 eSIM 配置文件或 VoIP 账号），以便在支持的设备上使用。运营商通过该虚拟号码计算实际使用量。
- 数据来源：平台通过后端系统接入合作运营商提供的流量使用数据接口（API）。
- 账单生成：平台后端每月月底接收来自运营商的当月服务费用账单。后端根据运营商账单数据，计算用户的最终应付费用，并根据平台管理员设置的手续费率，在运营商费用基础上加上平台手续费。
- 计费隔离：在后端和合约层面，运营商费用和平台手续费需明确区分存储和计算。用户界面将单独展示"运营商费用"和"平台手续费"类目，清晰透明。
- 未结账单处理：用户在次月开始时，如果存在未结清的账单，其服务状态将自动变更为"暂停"，不可继续使用平台服务。未结清账单的用户，其已存入的押金将持续锁定，直至所有账单结清。
- 账单确认：平台智能合约接收到用户支付后，自动更新用户账单状态，并将运营商费用转给运营商，平台手续费转入平台指定账户。
- Chainlink 增强：通过 Chainlink Functions 或 Data Feeds 获取运营商实时本地资费数据（若运营商提供 API），实现动态最优定价推荐；结合 Chainlink CCIP 未来可实现与运营商后端系统的安全跨链消息传递和结算确认。

### 3.3 去中心化身份（Decentralized Identity）

- 仅需钱包地址 + 保证金，无 KYC。

### 3.4 产品亮点与创新点（Key Features & Innovations）

- 完全去中心化 + Oracle 驱动：钱包地址即身份，链上信用 + 后付费 + 无 KYC。
- AI + Chainlink 双轮驱动：
  - 0G Compute 提供智能流量预测和推荐。
- 实时公平结算：Chainlink Price Feeds / Data Streams 确保多币种账单自动转换，透明且防操纵。
- 模块化架构：0G 全栈 + Chainlink Oracle/CCIP，形成高安全、可扩展的全球通信网络。

### 3.5 商业模式（Business Model）

- 主要收入：收取用户月付账单总额的 2.5% 作为平台手续费。
- 潜在收入：运营商分成、高级 AI 功能订阅、链上信用数据服务。
- 成本优势：0G 低成本存储与计算 + Chainlink 成熟的去中心化预言机服务，大幅降低中心化服务器和数据获取开销。

### 3.6 目标用户（Target Users）

- 数字游民、跨境工作者、短期旅行者、留学生。
- Web3 用户与加密爱好者（追求无 KYC、链上结算体验）。
- 任何需要灵活、透明跨境通信的全球用户。

### 3.7 技术实现路线（Technical Implementation）

- 区块链层：0G Chain 部署核心合约（保证金管理、计费逻辑、结算）。
- 数据层：0G Storage 存储使用日志和账单。
- 计算层：0G Compute 运行 AI Agents。
- 预言机层：
  - Chainlink Data Streams / Price Feeds：实时获取汇率数据，实现多币种动态结算。
  - Chainlink Functions：调用外部运营商 API 获取最新资费信息。
  - Chainlink CCIP：未来支持跨链消息传递、与多链运营商系统对接，以及跨链保证金/结算转移。
- 集成：eSIM/虚拟号码 API + 运营商网关 + 钱包连接。

#### 移动端功能（前端）

1. 钱包连接：通过 WalletConnect 连接主流 Web3 钱包。
2. 用户 Dashboard：显示当前账户状态（活跃/暂停）。
3. 显示当前押金余额及最低押金要求。
4. 显示当前的虚拟号码和绑定国家。
5. 实时或准实时显示流量/通话使用量。
6. 充值/提现：押金的存入和提取功能。
7. 虚拟号码管理：申请新虚拟号码、查看已申请号码。
8. 账单管理：显示当前未结账单、历史账单。
9. 账单详情显示（运营商费用、平台手续费）。
10. 账单支付功能。
11. 提前结账功能。
12. 通知中心：接收平台发送的账单通知、服务状态变更通知等。

#### 后端系统功能（Go）

1. 运营商集成层：与各个电信运营商的计费系统、虚拟号码管理系统、网络控制系统建立 API 对接。
2. 负责虚拟号码的申请、分配、激活、注销。
3. 负责获取用户实时或周期性的流量/通话使用数据。
4. 负责与运营商系统通信，实现服务（流量/通话）的实时或准实时启停。
5. 计量数据处理模块：从运营商集成层接收原始使用数据。
6. 数据标准化、聚合、校验。
7. 对处理后的数据进行数字签名。
8. 通过 Web3 客户端库调用链上的 MeteringOracleProxy 合约，提交使用量数据。
9. 账单生成与管理模块：定期（每月最后一天 UTC 12:00）从运营商集成层拉取用户账单数据。
10. 根据运营商费用和平台手续费率，计算用户最终账单。
11. 生成账单并将其数据存储在数据库中（包含运营商费用和平台手续费）。与链上 BillingManager 合约同步账单状态。
12. 通知服务：负责向用户发送账单通知、服务暂停通知、信用值变化通知等。
